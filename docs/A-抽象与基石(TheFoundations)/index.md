# 抽象与基石 (The Foundations)

这一部分用于收纳‘可迁移的基础工具’：形式化、证明、抽象建模、信息与系统观点。它的目标不是记结论，而是把可复用的推理框架沉淀下来。

## 这里关注什么
- 形式化表达：定义、符号系统、模型与公理化
- 证明与反例：从‘看起来对’到‘为什么对/在哪些条件下对’
- 抽象建模：把问题抽成结构（集合/关系/图/代数结构/概率模型）
- 信息与复杂性：信息量、压缩、通信、计算复杂度的直觉

## 本目录建议沉淀的内容
- 常用定义与等价刻画（尽量写‘等价条件’与‘反例’）
- 常用证明套路库：归纳、对角线、构造、极值、反证、交换求和等
- 你常用的符号约定与记号表（避免以后重复解释）
- 跨学科通用的‘抽象模板’：把问题映射到结构并复用定理

## 写作/学习提示（针对本主题）
- 遇到新概念时：写出 1) 定义 2) 最小例子 3) 非例子 4) 常见误解
- 遇到新定理时：记录它依赖的条件；写一个‘条件被破坏就失败’的例子
- 把每个主题的‘直觉解释’与‘形式版本’并排写，减少遗忘成本

## 边界（避免写到别处）
- 具体工程实现细节（放到‘技术与工程’）
- 具体学科的事实性百科罗列（放到对应学科目录）
