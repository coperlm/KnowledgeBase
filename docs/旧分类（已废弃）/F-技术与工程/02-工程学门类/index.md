# 工程学门类

工程学门类更适合沉淀‘工程方法论’：从需求到验证，再到失效与安全边界。具体学科知识可以按项目/主题细分。

## 这里关注什么
- 需求与约束：性能、成本、可靠性、法规、可制造性
- 设计与验证：建模、仿真、原型、测试与验收标准
- 工程管理：版本、变更控制、文档、可追溯性
- 安全与失效：FMEA/FTA、冗余、降级策略（高层次）

## 本目录建议沉淀的内容
- 项目模板：需求文档、接口定义、验证计划、测试报告结构
- 常见工程权衡记录（材料/工艺/成本/交期/可靠性）
- 失效案例库：失效模式—检测手段—缓解策略（高层次）

## 写作/学习提示（针对本主题）
- 写任何设计：先写验收标准与测量方法（否则很难闭环）
- 把“不可控的不确定性”写出来：环境、供应链、老化等
- 对复杂系统：分层写接口与责任边界，减少耦合
